一本五百页的书和我的二十六小时

五百页的书

我是一本书，名字叫 《HTML and CSS》，出自 Jon Duckett 之手，苛刻的豆瓣读者给了我九分，还不错。

这位叫文中昊的读者极少购书，能打动这样的铁公鸡，买上一本我这样的英文原版正版实体书，已说明我的气度不凡。

刚开始的一切都很美好，我被请回家中的时候正值夏天。他在躁动的夜晚为我宽衣解带。他的高中同桌传染给他一个怪癖：闻新书的墨香。于是我平躺在书桌上，将心扉袒露无余，他的指肚划过书页的时候，我会不由自主地发出沙沙的呻吟。他的鼻子凑过来，我能感受到鼻头上微沁的汗液。他吸气，我的味道混合着夏日潮湿的空气进入他的鼻腔。这让我很羞涩，因为这样的味道并非所有人都愿意接受。紧接着他缓缓的扬起头，双目微瞬，让我感受到他的陶醉。夜晚的微风带来一丝凉爽，温柔的灯光辉映着热情的凝望。

接下来的日子里，我将自己无所保留地呈现出来，他也叹服于我的美。版面精美绝伦，是为了骨架的清晰；字体精挑细选，是为了阅读的愉悦。每一章的不同部分采用统一的背景色搭配区分：只要我一个颜色，心意就被领会。话不多讲、字字玑珠。一个合格的尤物就要学会留白，在若即若离的暧昧里，给他空间又恰到好处地牵动他的心。迷人的外表之下还有这样一番渊博的胸襟，鲜有人不会臣服于我的扉页之下，无论雌雄。

但，好景不长，在差不多读到一半的时候，他开始朝三暮四，继而对我置之不理。虽然每过一段时间他又会重燃热情，但总是在大致相同的时候，再次将我打入冷宫。如此往复，直到有一天，我再也忍受不了这样的羞辱，愤然玉碎。

二十六个小时

我是文中昊，书翻散架的时候，我还没有完整地看完一遍。

书确实漂亮，以至于我时不时就会忍不住拿它来翻一遍。精美得不像一本技术读物，更像一本华而不实的画册。这也是我这个不怎么爱实体书的人愿意下手的原因。结构也好，清晰。且节奏合理，让人读起来没什么压力。

但技术书籍始终是技术书籍，这本书我能读到第九章已经不错了，每次再往下看的时候我就困、晕、想去厕所、玩手机、刷微博、然后被莫名奇妙的事情吸引。这样来来回回几遍，书居然散架了。

和好友袁隽谈及书读不下去的事，恰巧他也表示了相同的困惑。思来想去，我们打算共同读完电子版，以纪念我那本宁为玉碎不为瓦全的实体书。

第一天是在二零一六年的十一月二十八日，我们大致讨论了一下借助语音通话和屏幕共享来看这本书的方式，以及每天一小时的看书时间。

十一月二十九日第二天，我们看了三十页，掐指一算这样看下去，再加上周末如果每天能抽出八小时来看的话，我们两周就能读完啦！

十一月三十日日第三天，一小时读了二十三页，不错。

十二月三日，周末，我们高估了自己的定性，不仅没有做到周末每天八小时，连每天一小时都差点保不住。于是我们痛心疾首、痛改前非、痛定思痛地把周末每天看八小时的计划改成周末每天也是一小时。一方面古人云想嗨就嗨嗨得响亮，噢不，人生得意须尽欢莫使金樽空对月，另一方面连看八小时这种高中法西斯看书方式不能让它重演，不能重演，演... 男人就要对自己狠一点——那是有病。

另外这期间我们尝试了一天看俩小时：一天上班下来，再整两小时感觉被吸干了。所以觉得这事急不得，要结合自身的功力确定节奏，不然容易走火入魔，以后看见代码就吐怎么办，基于此秉承了每天一小时不变的原则。

十二月四日，只看了十页。开始就各种实例进行练习，因此进度放缓。练习很重要，有些例子看着没什么玄机，但真练起来就会发现各种搞笑的错误，比如漏了个分号结果下一行代码就不执行了。我们练习的时候还发现了书中好几个纰漏。因此我们不急不急，在这个全世界都要飞、讲求效率、恨不得上天的时候，我们坚信慢工出细活。

十二月九日，第二周过去了，看到了全书的一半多一点，比预期的两周看完一口吃个大胖子的目标慢了将近一倍，我们开始焦虑。另外随着学的东西多起来，前后联系越来越多，需要讨论的东西也相应增多，进度越来越慢，恐慌地想是不是抱团一起看也不行，也会虎头蛇尾变成一件遥遥无期的事儿。

且看的越多越发现啊原来写个网页这么麻烦，原来 HTML 和 CSS 就是冰山一角能看到的那部分，还有 python, javascript, SQL, PHP 等等很多东西啊，且不同的浏览器脾性不同，像 Internet Explorer 这种暴脾气非常难伺候。要不要继续看下去，是不是看点其它的像 python 或者转看 iOS 开发这种好像更有用？很焦虑。

十二月十日，第十五个小时，根据袁隽的意思：憋说话，看书，不论怎样先看完再说。

十二月十二日，第三个星期结束。我回了一次烟台，爸妈整的鲅鱼饺子、比目鱼、皮皮虾居然也没有让我放弃每天一小时的看书时间；他们看新闻联播、星光大道、健康之路、芝麻开门的时候我居然关起门来在写代码。简直就是富贵不能淫、威武不能屈。

这个时候看不看书是其次，你开始被自己的坚持感了一点动。

十二月十五日，第二十天。给自己简单写了一个简历网页，并按照黑底白字、内容为先、外观为辅的思路调成了自己最喜欢的样子。

十二月十八日，第二十三天。把所学用在了工作中，发广播邮件、写知识库的时候，调用 HTML。感觉就像黑客帝国的尼奥突然开窍了。

十二月二十三日，第二十六个小时，看得兴起，一个小时过去后两个人都意犹未尽。于是我们继续下去结果一不小心给读完了，读完了！

这本书真的非常适合入门，浅显的概念也会解释得周到，我觉得一个从未接触过 HTML 的中学生也能被顺利领上贼船，不，领进门。且思路正，鲜有边边角角用不到的知识。另外会穿插美工设计理论，最后一章甚至讲怎么吸引流量、怎么监测、怎么优化等等。

坦白讲我们两个对于 HTML 都稍有了解，因此这本书看了这么久其实有点慢。但转念一想一本五百多页的书连读带练带讨论，总共用了二十六个小时，效率也算可以了。

切磋有助于学习。小朋友总喜欢找个玩伴，即便弹个弹珠，一起玩也能玩出很多花样来。因此找到组织很重要。一个人的迷茫、困惑、怯懦、懒惰可以被两个人分担而减半；一个人的喜悦、成就感、灵感、经验却因两个人的分享而倍加。



二零一六年十二月二十四日，于上海

文中昊
